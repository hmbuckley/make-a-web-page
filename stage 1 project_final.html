Lesson 1: Basics of the Web and HTML

The World Wide Web

The web is made up of HTML documents(Hyper text markup language).It uses hyperlinks or links to glue it all together. It was created in the early 90s.The major pieces of the web are:

You, your computer and your web browser, the internet, and the servers that sit in wiring closets and host all of the different types of files, such as images, videos, documents etc.

HTTP(Hyper Text Transport Protocol)

HTML(Hyper Text Markup Language) is used to create web pages. HTML Markup uses elements that are made up of tags, content and attributes.

There are many differnt types of tags such as b for bold, em for italics, br for line break and p for paragraph.

You can insert links to other pages such as hockey to get the latest on the nhl.

Attributes have values such as nhl.com in the link above. All whitespace becomes a single line, so you can use br or p to create new lines and paragraphs.

Lesson 2: Creating a structured document with HTML

Tree Structure

When coding, it is good practice to use a tree structure to identify the diffeerent elements. Also when writing code it is good to use indents to keep your code easy to read.

All elements are boxes

When designing a web page start by identifying the big boxes and then working down to the smaller boxes.Sketch it out with a pencil.The div tag is used to create boxes. Use classes to label the boxes. For web pages HTML are the structure and Css is the style(colors, fonts, sizes).

Lesson 3 Adding CSS Style to HTML Structure

CSS (Cascading Style Sheets)

CSS is used to add style to your web page. When adding style to a document there are three differnet ways to include CSS code. You can put the code in the head section ( this works ok for small projects); write the style inline with your HTML (this is not recommended as it causes a lot of repeat coding); create a seperate css file that you link to(this is the best way to use CSS).

Re-using Code

When writing code you want to try to re-use as much as you can so that you are not continually re-writing the same code. The use of the div class name allows us to apply the same style to similar elements without having to re-write the code.

I must admit while creating this webpage I became the victim of cutting and pasting bad code as I was creating the different class names. The simple error of leaving out a leading " caused the same problem for each of my divs. This was a great lesson to learn.
